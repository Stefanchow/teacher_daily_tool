## 1. 项目背景分析

### 1.1 目标用户群体
- **主要群体**：K12阶段英语教师（覆盖小学、初中、高中）。
- **痛点**：备课时间紧张，教案标准化程度低，缺乏可视化教学辅助工具。

### 1.2 核心需求
- **自动化生成**：基于用户输入的“教学主题”，自动生成标准的30分钟或45分钟课时教案。
- **教学法适配**：系统必须内置并严格遵循 **PWP** (Reading/Listening) 和 **PPP** (Grammar/Vocabulary) 两种经典教学模式。

### 1.3 教学法支持概览
| 教学模式 | 适用课型 | 核心逻辑 |
| :--- | :--- | :--- |
| **PWP** | 阅读 (Reading) / 听力 (Listening) | Pre (准备) -> While (输入) -> Post (输出) |
| **PPP** | 语法 (Grammar) / 词汇 (Vocabulary) | Presentation (演示) -> Practice (练习) -> Production (产出) |

## 2. 教学法时长分配规范

### 2.1 PWP 模式 (Pre-While-Post)
> 适用于以**技能培养**为主的阅读和听力课程。

| 阶段 | 建议时长 | 环节细分 | 关键活动 |
| :--- | :--- | :--- | :--- |
| **Pre-Stage**<br>(导入阶段) | **10-15 min** | 激活背景知识 (5 min) | 头脑风暴、图片讨论、相关视频 |
| | | 预测内容 (3 min) | 标题/插图预测、提问引导 |
| | | 词汇预热 (5 min) | 核心障碍词汇扫除 |
| **While-Stage**<br>(核心阶段) | **20-25 min** | 泛读/泛听 (8 min) | 抓主旨 (Skimming)、寻读 (Scanning) |
| | | 精读/精听 (12 min) | 细节理解、长难句分析、逻辑推理 |
| | | 理解检测 (5 min) | 问答、判断正误、填空 |
| **Post-Stage**<br>(拓展阶段) | **10 min** | 讨论应用 (5 min) | 角色扮演、观点辩论、复述 |
| | | 拓展活动 (5 min) | 写作练笔、思维导图总结 |

### 2.2 PPP 模式 (Presentation-Practice-Production)
> 适用于以**知识习得**为主的语法和词汇课程。

| 阶段 | 建议时长 | 环节细分 | 关键活动 |
| :--- | :--- | :--- | :--- |
| **Presentation**<br>(呈现阶段) | **15 min** | 情境导入 (5 min) | 创设真实语境、引出目标语言 |
| | | 目标语言呈现 (7 min) | 规则讲解、例句展示、板书设计 |
| | | 概念确认 (3 min) | CCQ (Concept Checking Questions) |
| **Practice**<br>(练习阶段) | **20 min** | 控制性练习 (8 min) | 机械操练、替换练习 (Drills) |
| | | 半控制性练习 (7 min) | 填空、造句、看图说话 |
| | | 准确性反馈 (5 min) | 纠错、点评、强化记忆 |
| **Production**<br>(产出阶段) | **10 min** | 创造性应用 (7 min) | 任务型活动、情景对话、自由表达 |
| | | 流利度反馈 (3 min) | 针对流利度与交际效果的评价 |

## 3. 特色功能实现要求

### 3.1 思维导图数据生成 (Mind Map)
系统需输出符合前端渲染要求的 JSON 结构数据，满足以下标准：
- **主题关键词提取**：自动从文本中提取核心 Topic。
- **层级关系构建**：
    - Root: 单元/课程主题
    - Level 1: 核心板块 (如 Vocabulary, Grammar, Key Sentences)
    - Level 2: 具体知识点
- **可视化节点关联**：确保父子节点逻辑清晰，支持前端展开/折叠。
- **Mermaid 支持**：生成的思维导图数据需支持转换为 Mermaid Mindmap 语法，以便在 Markdown 教案中直接预览。

### 3.2 教学环节表格 (Lesson Plan Table)
生成可编辑的 Markdown 表格，需包含：
- **时间区块划分**：自动计算各环节起止时间。
- **教学法标注**：明确当前环节属于 PWP 或 PPP 的哪个阶段。
- **师生活动描述**：简述 Teacher Activity 和 Student Activity。

**Markdown 模板示例**：
```markdown
| 时间 | 阶段 | 教学环节 | 教师活动 | 学生活动 | 设计意图 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 00:00-05:00 | Pre | 导入 | 展示图片... | 观察并回答... | 激活背景知识 |
```
